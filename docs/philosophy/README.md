# Philosophy

> The Philosophy documents describe the principles, values and governance that guide the Urban Observation Platform (UOP).

Unlike the architecture documentation, which explains **what** the platform is and **how** it is implemented, the philosophy explains **why** the project exists and how it should evolve.

These documents provide the foundation for architectural and technical decision-making.

---

# Purpose

The philosophy documentation exists to:

- Define the long-term vision of the project
- Establish guiding principles
- Describe governance and decision-making
- Promote consistency across architecture and implementation
- Support sustainable open source development

The philosophy should remain stable over time and change only when the project's direction fundamentally evolves.

# Guiding Principles

The philosophy of UOP is based on a small number of enduring principles.

- Solve real problems.
- Prefer simplicity over unnecessary complexity.
- Keep architecture understandable.
- Preserve observations.
- Allow knowledge to evolve.
- Document significant decisions.
- Design for long-term sustainability.

These principles should guide every architectural and implementation decision.
